GitHubのファイルを他の人が編集できるようにする方法はいくつかあります。目的別にまとめます。

【方法①：Collaboratorとして招待する】
特定の人に直接編集してもらいたい場合。

手順
1. 対象のリポジトリを開く
2. Settings → Collaborators
3. Add people をクリック
4. GitHubユーザー名またはメールアドレスを入力
5. 相手が招待を承認すると編集（push）が可能になる

特徴
・Write権限以上で編集可能
・パブリック／プライベートどちらでも可

---

【方法②：Fork & Pull Request】
不特定多数や外部の人に編集してもらいたい場合。

流れ
1. 相手がリポジトリをFork
2. Fork先で編集
3. Pull Requestを作成
4. 管理者が内容を確認してMerge

特徴
・直接編集されないので安全
・オープンソース開発で一般的

---

【方法③：ブランチ保護を使う】
チーム開発向け。

例
・mainブランチは直接編集不可
・featureブランチは編集可
・Pull Request必須

設定場所
Settings → Branches → Branch protection rules

---

【方法④：GitHub Web上で編集】
コラボレーターの場合、ブラウザ上で直接編集可能。

手順
・ファイルを開く
・鉛筆アイコンをクリック
・編集してCommit

※ ForkしたリポジトリでもWeb編集は可能

---

【おすすめの選び方】
・知り合い1〜数人 → Collaborator
・不特定多数 → Fork & Pull Request
・チーム開発 → ブランチ保護 + PR
・文章中心 → Web編集
